# Airtype v2 — 產品需求文件（PRD）

> **版本**：2.0-draft
> **日期**：2026-03-21
> **狀態**：草稿

---

## 1. 產品願景

**一句話**：按下快捷鍵、說話、文字出現在游標位置。沒有其他步驟。

Airtype v2 是一次產品減法重構。v1 提供了 6 種 ASR 引擎、本地 LLM 潤飾、多種模型選擇——但使用者行為數據顯示：大多數人下載所有模型後隨便選一個，然後再也不碰設定。本地 LLM 潤飾的輸出品質不穩定，有時比原始辨識結果更差。

v2 的核心信念：**使用者只想要語音輸入，其他的他們不想管。**

---

## 2. 目標使用者

| 角色 | 情境 | 需求 |
|------|------|------|
| 一般辦公族 | Slack/LINE/Gmail/Teams 日常溝通 | 按下去說話就好，不想設定 |
| 開發者 | VS Code/Cursor/Terminal 中輸入 | 保留英文技術詞彙、自動 camelCase |
| 中文母語者 | 中英夾雜的日常對話 | 中英文混合辨識準確 |

---

## 3. 設計原則

1. **零設定啟動**：安裝完成 → 按快捷鍵 → 開始說話。不選模型、不選引擎。
2. **感知不到的存在**：浮動膠囊不搶焦點、不干擾工作流程、不彈視窗。
3. **視覺愉悅**：彩虹漸層波形動畫，讓語音輸入有「活著」的感覺。
4. **情境自動適應**：偵測前景應用程式，自動調整輸出格式，使用者不需要手動切換。

---

## 4. 與 v1 的差異對照

| 項目 | v1（現狀） | v2（目標） |
|------|-----------|-----------|
| ASR 引擎 | 6 種（ONNX/PyTorch/Vulkan/MLX/Sherpa/Breeze） | **僅 Qwen3-ASR ONNX** |
| 模型選擇 | 使用者手動選擇、手動下載 | **自動下載、零選擇** |
| LLM 潤飾 | 本地 GGUF + API 雙模式 | **移除** |
| 波形動畫 | 7 條單色柱狀圖 | **彩虹漸層流動曲線** |
| 情境感知 | 無 | **偵測前景應用 → 規則式後處理** |
| 設定頁面 | 模型管理/引擎選擇/LLM 設定/外觀/快捷鍵/詞典/語音/關於 | **快捷鍵/詞典/關於**（大幅精簡） |
| 打包體積 | ~500MB+ | **目標 < 200MB** |
| 平台優先序 | macOS/Windows 並行 | **Windows 優先，macOS 後續** |

## 5. 功能規格

### 5.1 核心功能：語音轉文字

#### 5.1.1 ASR 引擎

- **唯一引擎**：Qwen3-ASR 0.6B（ONNX Runtime CPU）
- **模型來源**：andrewleech/qwen3-asr-onnx（HuggingFace）
- **推理方式**：ONNX Runtime，純 CPU，無 GPU 依賴
- **語言支援**：中文、英文、中英混合
- **首次啟動**：自動下載模型至 `~/.airtype/models/`，顯示下載進度

#### 5.1.2 語音活動偵測（VAD）

- 保留現有 Silero VAD 實作
- 自動偵測說話開始/結束
- 可設定靜音超時時間（預設 1.5 秒）

#### 5.1.3 文字注入

- 辨識完成後自動將文字輸入至游標位置
- 支援剪貼簿模式（`Ctrl+V`）與模擬鍵盤模式
- 保留現有 `text_injector.py` 實作

---

### 5.2 浮動膠囊 UI

#### 5.2.1 外觀

- **形狀**：膠囊形（pill shape），圓角 = 高度 / 2
- **尺寸**：寬 220px × 高 48px（可縮放 0.8–1.5x）
- **背景**：深色半透明（#0a0a0a，92% 不透明度）
- **不搶焦點**：`WS_EX_NOACTIVATE`（Windows）/ `Qt.Tool`

#### 5.2.2 彩虹漸層波形動畫

取代現有的 7 條單色柱狀圖，改為連續流動曲線：

- **資料點**：20–30 個取樣點，三次貝茲插值產生平滑曲線
- **顏色**：水平方向 `QLinearGradient`，色帶為粉紅→橙→綠→藍→紫
- **流動效果**：漸層色帶偏移量隨時間遞增，產生左→右流動感
- **RMS 驅動**：曲線振幅由麥克風 RMS 音量驅動
- **對稱**：上下鏡像繪製（中軸對稱）
- **更新率**：≥ 30 FPS（33ms QTimer）
- **閒置狀態**：微弱呼吸動畫（低振幅正弦波）

#### 5.2.3 狀態指示

簡化為 3 種視覺狀態（不再顯示狀態文字）：

| 狀態 | 波形行為 | 背景 |
|------|---------|------|
| 閒置 | 靜止，僅顯示膠囊外殼 | 深色 |
| 聆聽中 | 彩虹波形隨音量跳動 | 深色 |
| 處理中 | 波形變為脈動呼吸動畫 | 深色 |

#### 5.2.4 控制元素

- **麥克風按鈕**：點擊切換錄音開始/停止（QPainter 自繪圖示）
- 移除音訊裝置選擇下拉（自動使用系統預設麥克風）
- 移除狀態文字標籤
- 移除即時預覽文字（簡化視覺）

#### 5.2.5 動畫

- 滑入：200ms slide-up + fade-in（OutCubic 緩動）
- 滑出：150ms slide-down + fade-out
- 位置記憶：拖曳後自動儲存

---

### 5.3 情境感知輸出（Context-Aware Output）

#### 5.3.1 前景應用偵測

```
Windows API: GetForegroundWindow() → GetWindowText() / GetClassName()
```

偵測頻率：每次觸發語音輸入時偵測一次（非持續輪詢）。

#### 5.3.2 應用程式情境規則

| 應用程式類別 | 偵測方式 | 後處理規則 |
|-------------|---------|-----------|
| **通訊軟體**（Slack/LINE/Discord/Teams） | 視窗標題或程序名比對 | 預設模式，ASR 直出 |
| **郵件**（Gmail/Outlook — 瀏覽器標題） | 瀏覽器標題包含 `Gmail` / `Outlook` | 加入標點符號、斷句優化 |
| **程式編輯器**（VS Code/Cursor/JetBrains） | 程序名比對 | 開發者模式 |
| **終端機**（Windows Terminal/PowerShell/CMD） | 程序名比對 | 命令模式 |
| **其他** | 預設 | ASR 直出 |

#### 5.3.3 開發者模式規則（規則式，不需要 LLM）

- 偵測到英文多詞短語 → 自動轉 `camelCase`（例：「get user by id」→ `getUserById`）
- 偵測到常見 CLI 指令模式 → 保持原樣不加標點（例：「git commit dash m」→ `git commit -m`）
- 保留技術專有名詞大小寫（透過內建詞典：`API`、`HTTP`、`JSON`、`React`、`Docker` 等）
- 不自動加句號

#### 5.3.4 標點符號規則

- **通訊模式**：不加句號（口語習慣）
- **郵件模式**：自動斷句、加句號逗號
- **開發者模式**：不加標點
- **命令模式**：不加標點

- **預設**：`Ctrl+Shift+Space`（可自訂）

  - 按下 → 啟動錄音 → 膠囊滑入

- 保留現有 `pynput` 實作

## 11. 開放問題

| 問題 | 備註 |
|------|------|
| 即時預覽是否完全移除？ | 串流辨識的即時預覽對使用者有幫助，但增加視覺複雜度。可考慮保留為隱藏選項。 |
| 郵件模式的標點斷句用規則能做到多好？ | 純規則式可能不夠自然，可能需要簡單的統計模型或未來開放雲端 API 選項。 |
| 開發者模式的 camelCase 觸發條件？ | 需要明確定義：連續英文單詞幾個以上才觸發？單個單詞怎麼處理？ |
| 模型自動下載失敗的回退方案？ | 離線安裝包？手動下載連結？ |
| 是否保留 ASR 引擎抽象層？ | 即使只剩一個引擎，Protocol 抽象可能有利於未來擴展。建議保留但簡化。 |
