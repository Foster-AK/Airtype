## 背景

辭典可提升 ASR 在專業領域的辨識精度。PRD §5.5 定義了熱詞、替換規則及辭典集。

相依性：06-asr-abstraction、16-settings-panel。

## 目標 / 非目標

**目標：**

- 帶有權重（1-10）的熱詞，傳遞至 ASR 引擎
- 於 ASR 輸出後套用替換規則（字串與正規表達式）
- 辭典集（具名群組、多選啟用）
- 編輯、匯入/匯出的 UI 介面

**非目標：**

- 不修改 ASR 引擎本身（熱詞透過現有 Protocol 注入）

## 決策

### 辭典引擎作為管線的後處理器

替換規則在 ASR 輸出後、LLM 潤飾前套用。熱詞透過 `set_hot_words()` 設定至當前啟用的 ASR 引擎。

### 辭典檔案使用 JSON 儲存

每個辭典集以 JSON 檔案形式存放於 `~/.airtype/dictionaries/`。格式：`{hot_words: [...], replace_rules: [...]}`。應用程式隨附預設辭典集。

### 辭典 UI 使用可編輯表格元件

使用 QTableWidget 編輯熱詞與替換規則。支援行內編輯、新增/刪除列、逐條啟用/停用。

## 風險 / 取捨

- [風險] 正規表達式替換規則可能影響效能 → 緩解措施：預先編譯正規表達式；規則數量限制為 100 條
- [取捨] JSON 格式簡單，但不適合大型辭典 → 對於典型使用情境（< 1000 筆）可接受
