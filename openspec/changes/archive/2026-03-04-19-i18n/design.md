## 背景

Airtype UI 支援 4 種語言：zh-TW（預設）、zh-CN、en、ja。PRD §5.2 明確規定需支援執行期間語言切換。

相依性：16-settings-panel。

## 目標 / 非目標

**目標：**

- 為所有 UI 字串提供翻譯系統
- 在不重新啟動的情況下於執行期間切換語言
- 建立 4 個語言檔

**非目標：**

- 不切換 ASR 語言（那是 ASR 引擎的工作）
- 不翻譯日誌訊息或錯誤代碼

## 決策

### JSON 架構的翻譯檔

在 `locales/` 目錄中存放簡單的 `{key: translated_string}` JSON 檔案。`tr(key)` 函式查詢當前語言的對應字串，若找不到則退回 zh-TW。

**不採用 gettext/Qt Linguist 的原因**：管理更簡單；JSON 可直接人工編輯；無需編譯步驟。

### 以 Signal 為基礎的語言變更通知

當語言改變時，發出一個 Qt Signal。所有顯示文字的 UI 元件連接此 Signal，並於收到通知後刷新其標籤。

## 風險 / 取捨

- [風險] 缺少翻譯時會顯示 key 本身 → 緩解措施：退回鏈：請求語言 → zh-TW → key 本身
- [取捨] 需手動在 tr() 中包裹字串 → 此為必要做法；自動化萃取易出錯
