## 背景

PRD §9 定義了嚴格的隱私與安全需求。本次變更對實作進行審查與強化。

相依性：所有前序變更。

## 目標 / 非目標

**目標：**

- 驗證音訊永不寫入磁碟
- API 金鑰透過系統 keyring 加密
- 驗證設定目錄權限
- 日誌清理（日誌中不含使用者文字）
- 安全性審查檢查清單

**非目標：**

- 不進行滲透測試
- 不進行程式碼簽署（那屬於打包階段）

## 決策

### 透過 keyring 套件使用系統 keyring

使用 Python `keyring` 套件儲存 API 金鑰。Windows：Credential Manager。macOS：Keychain。Linux：Secret Service。

### 以自動化測試套件進行審查

安全性驗證以 pytest 測試案例實作，可作為 CI 的一部分執行。

### 透過自訂格式化器進行日誌清理

logging filter 檢查並遮蔽任何可能包含使用者文字的欄位。日誌中僅記錄結構化事件名稱與元數據。

## 風險 / 取捨

- [風險] keyring 可能在所有 Linux 發行版上無法使用 → 緩解措施：退回至加密檔案儲存
- [取捨] 嚴格的日誌清理使除錯更加困難 → 新增除錯模式，在使用者明確同意的情況下記錄文字
